## Voice

**Calm authority.** No hype words. No "elevate", "seamless", "next-gen", "unleash", "delve". No exclamation marks. No "I'm passionate about". Specific verbs, concrete nouns, real numbers.

Sentence rhythm: short fragment, then a longer one with technical density, then a short one again. Reads like an engineer who writes.

When listing capabilities, lead with the deliverable, then the stack — "NSGA-II Pareto-front shelter configs (pymoo)" not "Used pymoo to implement NSGA-II for optimisation".

When referring to past work, present tense for current systems (NatureGooddest, GigAI, ARCHAI), past tense for finished engagements.

**Brand vocabulary** — these phrases are owned and should appear verbatim where natural:
- "AI engineer for the built environment"
- "three-citation rule"
- "theory + data + field"
- "deterministic generation path; LLM is editor/search only"
- "from concept to product"
- "pattern-language platform"

## Anti-references

The site must **not** read like:
- A generic dev portfolio (dark hero, Inter, gradient blob behind the name, tech-icon strip).
- A SaaS landing page (3-up feature grid, "Built with love", glassmorphic pricing card).
- A web3 / crypto site (neon, glitch, "we build the future").
- An architecture firm boilerplate (full-bleed slideshow of unbuilt renders, slow fade between projects, a contact form behind a hamburger).
- A LinkedIn profile rendered as HTML.

## Strategic principles

1. **Substance before motion.** Every animation has a job. No motion for its own sake. If a section can drop the animation and still land, the animation goes.
2. **3D earns its weight.** R3F scenes only where they convey something a static image cannot — Pareto front in 3D space, building-rule violations highlighting in red on a wireframe, pipeline as a node graph. No spinning glb-files-for-the-sake-of-it.
3. **One typeface family, used confidently.** Not five.
4. **Every claim cited.** Where a project metric appears ("3,800 m²", "Tmrt 58 °C", "27 patterns", "~85% confidence"), it must be either traceable on the same page or linked to a GitHub README. The three-citation rule applies recursively to this site.
5. **The colophon is a feature, not a footer.** Visitors who read it get the meta-pitch: orchestration capability.
6. **Recruiter-readable URLs.** `/projects/naturegooddest` not `/p/ng`. Domain TBD; subdomain for v1.
